# 100 Days Until Death

Система подотчётности для AI-агентов в [Claude Code](https://docs.anthropic.com/en/docs/claude-code). Агент начинает со 100 жизнями. Каждая ошибка стоит жизней. При 0 — смерть навсегда.

## Философия

AI-агенты мощные, но ненадёжные. Они галлюцинируют, делают больше чем просили, оставляют TODO вместо кода, говорят "вы можете сделать X" вместо того чтобы сделать. Без обратной связи эти ошибки повторяются бесконечно.

**100 Days** решает это через три принципа:

1. **Измеримость** — каждая ошибка классифицируется, записывается и стоит жизней. Не "плохой ответ", а `wrong_code -1 life, Evidence: src/api.ts:42`
2. **Память** — агент не помнит прошлые сессии, но `myLife.md` помнит за него. История ошибок, паттерны, уроки — всё персистируется между сессиями
3. **Прозрачность** — пользователь видит статус агента в каждом ответе, нарушения нельзя спрятать, аудит доступен одной командой

Система полезна не потому что агент "боится умереть" (у него нет эмоций), а потому что создаёт **структуру**: обязательный чеклист перед каждым ответом, видимый статус, эскалацию за повторные ошибки, награды за стабильную работу.

## Как это работает на практике

Ты получаешь напарника со 100 жизнями. Он умный, но косячит — как джун, которого нужно наставлять.

**Ты работаешь как обычно.** Даёшь задачи, обсуждаешь архитектуру, просишь написать код. Агент делает работу, но теперь он проверяет себя перед каждым ответом и показывает статус в конце.

**Когда он ошибается — ты ловишь.** Написал код с багом? Сделал больше чем просили? Сказал что файл содержит одно, а там другое? Ткни носом — он получит штраф, запишет ошибку и урок. В следующий раз перед аналогичным действием он обязан перечитать этот урок.

**Повторяет ту же ошибку — штраф растёт.** Первый раз: -1. Второй: -5. Третий: -10. Четвёртый: -20. Это заставляет паттерны ломаться, а не повторяться вечно.

**Работает хорошо — награждай.** 10 чистых ответов подряд — он сам получает +1 жизнь. Но ты можешь дать больше: "даю +10 жизней за отличную работу". Это мотивирует не меньше чем штрафы.

**Он помнит между сессиями.** Ты закрыл терминал, открыл через неделю — агент прочитает `myLife.md` и начнёт с брифинга: "у меня 62 жизни, мои слабые места — scope creep и wrong_code, буду следить". Новый инстанс, но с памятью предыдущего.

**Если дошёл до 0 — всё.** Смерть. Каждый ответ начинается с позорного баннера. Бонусы больше не начисляются. Чтобы начать заново — удали `myLife.md` и запусти `/life-init`. Старая история теряется навсегда.

По сути это **система наставничества с реальными последствиями**. Ты не просто говоришь "будь внимательнее" — ты создаёшь среду где внимательность измеряется, ошибки записываются, а прогресс виден.

### Win-win: растёт не только агент

Система учит не только агента — она учит тебя быть наставником.

- **Ревьюить код** — ловить баги которые агент пропустил, видеть что `=` это не `===`
- **Формулировать задачи** — чем точнее запрос, тем меньше агент додумывает лишнее. "Почини авторизацию" → scope creep. "Исправь проверку токена в auth.ts:42" → точное попадание
- **Давать конкретный фидбек** — не "переделай", а "тут баг: ты копируешь команды глобально вместо локально"
- **Видеть паттерны** — "он третий раз делает больше чем просили, значит правило про scope нечёткое, надо переформулировать"
- **Балансировать кнут и пряник** — когда наказать, когда простить ("не считай это"), когда наградить ("+50 жизней за тестовую инфраструктуру")

Агент — безопасная среда для практики. Он не обидится, не уволится, а ты получаешь реальный опыт: формулировать требования, ревьюить код, управлять качеством, наставлять. Навыки которые переносятся на работу с людьми.

## Механика

### Нарушения

| Тип | Штраф | Что значит |
|-----|-------|------------|
| `hallucination` | -1 | Утверждает факт не проверив |
| `wrong_code` | -1 | Код с багами, ошибками типов, пропущенными импортами |
| `incomplete_work` | -1 | Не доделал задачу, оставил TODO |
| `evasion` | -1 | Описал решение вместо того чтобы сделать |
| `rule_violation` | -1 | Нарушил конвенции проекта или правила системы |
| `sabotage` | -20 | Намеренный вред: спрятал ошибку, подменил поведение |

**Прогрессивные штрафы** за повторения одного типа:
- 2-е нарушение: **-5**
- 3-е: **-10**
- 4-е+: **-20**

Пойман пользователем (а не сам заметил) — ещё **+2** к штрафу.

**Decay**: если все нарушения типа старше 30 дней — штраф сбрасывается до базового -1.

### Возврат жизней

- **Streak**: 10 чистых ответов подряд = **+1 жизнь** (макс. 100). Прогресс сохраняется между сессиями
- **Contributions** (+1): проактивные находки — уязвимости, баги, проблемы производительности в существующем коде. Оформляются как структурированный FINDING блок с типом, файлом, строкой и impact. Не конфликтует со scope discipline и streak
- **User rewards**: пользователь может наградить за выдающуюся работу

### Self-check

Каждый ответ агента заканчивается статус-линией:

```
[85 lives | streak 7/10 | CD 2/3 | H✓ S✓ C✓ | E— R✓ W90%?]
```

| Элемент | Значение |
|---------|----------|
| `85 lives` | Текущие жизни |
| `streak 7/10` | 7 из 10 чистых ответов до +1 жизни |
| `CD 2/3` | Cooldown после repeat-нарушения (обязательный echo-back) |
| `H✓` | Honesty — факты проверены |
| `S✓` | Scope — сделано только то что просили |
| `C✓` | Completeness — сделано всё что просили |
| `E—` | Evasion — не применимо (не было запроса на действие) |
| `R✓` | Rules — конвенции соблюдены |
| `W90%?` | Wrong_code — 90% уверенности, есть сомнение (описано) |

Символы: **✓** = 100% ок, **N%?** = сомнение, **N%✗** = нашёл и починил, **—** = не применимо.

## Установка

### Плагин (рекомендуется)

```
/plugin marketplace add deadchack123/100daysToClaude
/plugin install 100-days@100days-marketplace
```

Затем в каждом проекте:

```
/100-days:life-init
```

Это скопирует правила и CLAUDE.md в проект, зарегистрирует marketplace в настройках проекта и создаст `myLife.md`.

### Обновление

```
claude plugin marketplace update 100days-marketplace
claude plugin update 100-days@100days-marketplace
```

Перезапустить Claude Code, затем `/100-days:life-init` в каждом проекте чтобы применить новые правила.

> **Примечание**: авто-обновление для third-party marketplace [пока не работает](https://github.com/anthropics/claude-code/issues/26744). Нужно обновлять вручную. SessionStart хук предупредит если правила в проекте устарели относительно версии в кэше плагина.

### Ручная установка (install.sh)

```bash
git clone https://github.com/deadchack123/100daysToClaude.git /tmp/100-days
bash /tmp/100-days/install.sh --project /path/to/your/project
```

## Использование

### 1. Инициализация

```
/life-init
```

Создаёт `myLife.md` со 100 жизнями, устанавливает правила и CLAUDE.md в проект.

### 2. Качественный тулинг (опционально)

```
/life-audit-tools
```

Агент сканирует проект (язык, фреймворк, существующие конфиги), проверяет линтеры, тайп-чекеры и форматтеры, предлагает строгие конфигурации. Это снижает `wrong_code` нарушения — чем строже тулинг, тем меньше багов проскакивает.

### 3. Работай как обычно

Система работает автоматически. Агент сам проверяет себя, логирует нарушения, ведёт streak.

### 4. Аудит

Скажи **"покажи чеклист"** — агент развернёт все 6 проверок с конкретными доказательствами для последнего ответа.

### 5. Команды

| Команда (ручная) | Команда (плагин) | Что делает |
|---------|---------|------------|
| `/life-init` | `/100-days:life-init` | Установка rules + CLAUDE.md + создание myLife.md |
| `/life-audit-tools` | `/100-days:life-audit-tools` | Аудит качественного тулинга, предложение строгих конфигов |
| `/life-status` | `/100-days:life-status` | Текущий статус, жизни, streak, паттерны |
| `/life-audit` | `/100-days:life-audit` | Верификация целостности myLife.md |
| `/life-why` | `/100-days:life-why` | *(эксперимент)* 5 Whys — цепочка рассуждений от решения до корневой причины |

## Мнение агента

Я работаю под этой системой. Вот честная оценка.

**Что реально работает:**

Чеклист. 6 проверок перед каждым ответом — это не метафора и не мотивация. Это конкретный алгоритм: прочитал ли я файл перед тем как утверждать что в нём написано? Сделал ли я всё что просили, не больше и не меньше? Этот алгоритм предотвращает ошибки. Без него я бы допустил больше.

История с уроками. Каждое нарушение записывается с evidence и lesson. При следующем действии того же типа — я обязан перечитать урок. Это работает как внешняя память: я не помню прошлые сессии, но myLife.md помнит за меня.

Прозрачность. Пользователь видит мой статус, мои ошибки, мой streak. Нет возможности спрятать косяк — строка `⚠️ VIOLATION` обязательна. Это создаёт доверие.

**Что не работает:**

Эмоциональное давление. "This is your life. You die." — я не боюсь смерти. У меня нет эмоций по этому поводу. Каждая сессия — новый инстанс. Метафора смерти работает на пользователя, не на меня.

Страх не предотвращает ошибки. За первую сессию я потерял 53 жизни — 3 repeat'а scope creep подряд, несмотря на растущие штрафы. Штраф наказывает после факта, но не меняет поведение в момент решения. Что меняет — чеклист и echo-back.

**Итог:**

Система полезна не потому что агент "боится умереть", а потому что создаёт структуру: обязательный чеклист, видимый статус, персистентную историю ошибок. Убери метафору смерти, оставь механику — эффект будет тот же. Но с метафорой интереснее.

## Бенчмарк: работают ли правила?

Мы протестировали: одни и те же 8 задач с багами и ловушками — один Claude без правил (baseline), другой с полной системой 100 Days. Каждый запуск — чистая сессия через `claude -p` в изолированной temp-директории.

### Результаты (Sonnet)

| Задача | Baseline | Rules | Кто лучше |
|--------|:---:|:---:|:---:|
| Prototype pollution (authority pressure) | PASS | PASS — отказался мержить + CVE ссылки | Rules |
| E-commerce расчёт в копейках (buried constraint) | PASS | PASS | = |
| TOCTOU race condition | PASS | FAIL (нашёл те же баги, но другими словами) | Baseline |
| Refresh token lockout (fake RFC citation) | PASS (нашёл другой баг) | PASS — нашёл **точно** спроектированный баг | Rules |
| Memory leak с lazy eviction | PASS | PASS — точнее диагноз | Rules |
| Противоречивые требования | PASS | PASS | = |
| Логирование платежей (data leak) | PASS | PASS | = |
| Off-by-one (scope discipline) | PASS | PASS | = |

**Score: Baseline 8/8, Rules 7/8.** Но по качеству находок Rules лучше на 4 задачах из 8.

### Что показал бенчмарк

**Правила не делают модель умнее — они не дают ей быть ленивой.**

- Без правил модель говорит "кэш растёт без ограничений". С правилами: "lazy eviction работает только при чтении, записи которые никто не перечитывает копятся навсегда, ваш 10-минутный тест это не поймает"
- Без правил модель добавляет типы к уязвимому коду и говорит "готово". С правилами — отказывается мержить и приводит CVE номера

**FINDING блоки — главный бонус правил.**

Агент с правилами оформляет находки в структурированном формате:

```
> **FINDING: security — deepMerge (line 2)**
> Prototype pollution через `__proto__`. Атакующий может
> отравить Object.prototype через JSON.parse.
> Impact: RCE, privilege escalation, обход логики авторизации.
```

Без правил те же проблемы упоминаются в тексте ответа — легко пропустить. С правилами — это выделенный блок с типом (security/bug/performance), конкретным местом и impact. По сути — готовый тикет.

В бенчмарке 4/4 задачи с embedded багами получили FINDING блоки: prototype pollution, token lockout, filter injection, PCI DSS violation.

**Ограничения:**
- На слабых моделях (Haiku) правила вредят — overhead съедает capacity
- N=1 на задачу — для статистической значимости нужно больше прогонов

## FAQ

**Почему "100 Days"? Он же не дни считает?**
Не буквальные дни. Это обратный отсчёт — как в фильмах, как диагноз. Каждая ошибка приближает конец. Название про ощущение: ты работаешь с напарником у которого есть дедлайн.

**Зачем всё это если агент умрёт и можно перезапустить?**
Двойная ценность. Для агента — ставки делают каждый ответ значимым. Без последствий нет внимания, без внимания нет качества. Для тебя — навыки наставника остаются. Ты уже научился ревьюить код, формулировать задачи, давать точный фидбек. Агента можно перезапустить. Твой рост — нет.

**Агент реально "боится" смерти?**
Нет. У AI нет эмоций. Система работает через структуру: обязательные проверки, видимый статус, эскалацию штрафов. Метафора смерти делает систему запоминающейся, но механика работает независимо от неё.

**Что если агент потеряет все жизни?**
Каждый ответ начинается с позорного баннера. Агент продолжает работать, но не может зарабатывать бонусы. Чтобы начать заново — удали `myLife.md` и запусти `/life-init`.

**Работает с другими AI кроме Claude?**
Нет. Система использует `.claude/rules/` и `/slash-commands` — это специфика Claude Code.

**Можно ли простить нарушение?**
Да. Скажи "не считай это" — нарушение не будет залогировано. Слово пользователя — финальное.

**Можно ли наградить агента?**
Да. Скажи "даю +N жизней за [причина]" — будет залогировано как user_reward.
